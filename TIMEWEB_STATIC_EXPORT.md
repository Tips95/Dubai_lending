# Настройки для Timeweb Cloud - Статический экспорт

## Важно! Настройки в панели Timeweb Cloud:

### 1. Тип приложения:
- **Тип**: Статический сайт (Static Site) или Frontend
- **НЕ используйте**: Next.js или Docker (они ожидают сервер)

### 2. Директория сборки:
- **Директория сборки**: `out` (папка со статическими файлами)
- **Или**: `.` (корень, если файлы в корне)

### 3. Команда сборки:
- **Команда сборки**: `npm run build`
- После сборки создается папка `out` с файлами

### 4. Команда запуска:
- **НЕ нужна** (для статического сайта)
- Или оставьте пустой

### 5. Проверка:
После сборки в папке `out` должен быть файл `index.html`

## Что изменилось:

1. `next.config.js` - добавлен `output: 'export'` для статического экспорта
2. `images.unoptimized: true` - для статического экспорта
3. После `npm run build` создается папка `out` со всеми статическими файлами

## Структура после сборки:

```
out/
  ├── index.html          ← Главная страница
  ├── _next/
  │   ├── static/
  │   └── ...
  └── ...
```

## Важно:

Timeweb Cloud ищет `index.html` в корне директории сборки. 
Теперь он будет в папке `out/index.html` - укажите `out` как директорию сборки.
